A (static) website consists of 3 parts:
  • HTML
    •
  • CSS
    •
  • JS
    •
The HTML is the file that is first read by the browser. This means the browser (IE, Firefox, Chrome, Safari, Opera) will read this file and figure out how to display the text. Think of this as the skeleton of your website. This contains all the words and images you want to display.

If you're looking at the file in its raw format now (not in the browser), you might be wondering why it's indented. Does indenting even mean anything? Nope! not at all. It's just to make it easier to read*. If you wanted to, you could have this all in 1 line, and the browser will still be able to generate the same website from your instructions. But it'd be a pain to edit later on!

*If there is weird indenting in this file, forgive me, for I made this site in Notepad. I swear I'm more competent than this formatting would suggest. But have you ever tried doing this without syntax highlighting? Losing your own divs and doing your own indenting is really hard.
The CSS is the file that will add fonts and styling to your html. Try loading the file without CSS, and you'll see what a giant difference it makes to the aesthetics of your site!

How do you apply CSS where you want to? If you say 'font:14px', how do you target it to a specific text? Ask your mentor about IDs and classes.
JavaScript is designed to make your site do more fancy things, like animations. For example, you can make the text black, pop up an alert box, or make a turtle fly across your screen. A lot of times, you don't need JS to make your website spiffy if your CSS is enough. As you can see from making the text blue, there is some overlapping utility between JS and CSS, and depending on the situation, you might find it easier to use one or the other.
What's next? Now that you know the basics, there's so many directions web design can take you. Web design is great because the iteration process is fast. Your results are instantly displayed without needing to wait for a whole hour for your neural network to train, and then figuring out you wrote it wrong, and then having to fix it and train it again. Web design can lead you to careers such as a graphic designer, front-end engineer, a UX/UI designer, or an artist. Ask your mentor about these career paths to learn more!
